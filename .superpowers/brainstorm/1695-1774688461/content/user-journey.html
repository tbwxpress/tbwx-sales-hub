Who uses the Sales Hub — and what do they need?

Before we design anything, let's nail the user journey. Two types of users, very different needs.

👑
Admin (Gavish)
Runs the business
Morning check-in
How many new leads came in? Who replied? Any hot leads ready to close?
Mid-day oversight
Are agents following up? Is the pipeline moving? Any stale leads?
End of day
How many converted this week? Agent performance review.
🎯
Sales Agent
Closes deals every day
Opens app → instant confusion 😟
Currently: loads dashboard with stats. Agent thinks "ok... now what?"
Needs to know: who to call NOW
Leads that replied, leads that are hot, leads with follow-ups due today.
Needs to know: what to say
Quick reply suggestions, templates, last conversation context — all in one click.

The core UX problem to solve 👇

A

Action-First Dashboard

When an agent logs in, they see: "You have 3 replies waiting · 2 follow-ups due today · 1 HOT lead to call" — with direct action buttons. No hunting. Zero confusion.

B

Smart Inbox-First

Landing page IS the inbox/task list. Replies, follow-ups, and hot leads all in one unified feed — sorted by urgency. Agent just works down the list. Like email but for leads.

C

Both — Role-Aware Landing

Admin lands on stats/overview. Agent lands on their action queue. Same app, different default view based on role. Everyone sees exactly what they need first.
